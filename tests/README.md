# Test layout

Rust unit tests live beside the domain, components, infrastructure, and image-processing modules they exercise. Frontend unit/component tests live under `src/lib` and run with Vitest. Phase 4 coverage includes provider parsing, typed capabilities, factories, registry switching/fallback, persisted settings, timeouts, placeholder errors, manifest validation, local model metadata discovery, diagnostics, component UI, explicit connection testing, and performance formatting. This directory remains reserved for future end-to-end fixtures and Tauri integration tests.
